## 1. 本次目标

V1.2B 的目标是新增一个可独立运行的 openWakeWord demo 入口，用来验证后续 Wake Engine 接入前的本机可行性：

- `scripts/wake_engine_demo.py --help` 不依赖 openWakeWord。
- `--check-install` 能检查 optional dependency，缺失时不崩溃。
- `--dry-run` 不加载模型、不打开麦克风。
- `--list-devices` 尽量列出麦克风设备。
- 实时监听路径只在 openWakeWord 和音频后端都可用时运行。

本阶段不修改 `voice_overlay.py`，不修改 wake/session/TTS/LLM 主链路，不启动 STT server，不下载模型，不训练中文“贾维斯”模型。

## 6. 是否可列麦克风设备

可以。`scripts/wake_engine_demo.py --list-devices` 通过 `sounddevice` 列出输入设备，当前可见 12 个 input device。包含：

- index `0`：Microsoft 声音映射器 - Input
- index `1` / `5` / `9`：麦克风阵列
- index `13`：Realtek HD Audio Mic input
- index `20`：麦克风阵列 3，默认采样率 16000

注意：设备名包含 Windows 驱动返回的特殊字符，demo 已将 stdout/stderr 设置为 encode errors replace，避免 GBK 控制台编码崩溃。

