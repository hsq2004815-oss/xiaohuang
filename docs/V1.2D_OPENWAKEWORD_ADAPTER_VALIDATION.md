## 1. 目标

V1.2D-A 的目标是把 V1.2B demo 中的 openWakeWord runtime 逻辑收敛到独立 adapter，验证 optional import、状态报告、start/stop 生命周期、event callback、cooldown 合并和 demo 复用路径。

本阶段不替换当前 STT 文本唤醒，不启动 STT server，不调用 LLM/TTS，不训练中文“贾维斯”模型。

## 2. Adapter 文件和职责

新增文件：

```text
src/xiaohuang/openwakeword_adapter.py
```

职责：

- `OpenWakeWordDependencyStatus`：结构化描述 openwakeword / numpy / sounddevice / onnxruntime 可用性。
- `check_openwakeword_dependencies()`：只做 optional import 检查，不打开麦克风，不加载模型。
- `OpenWakeWordAdapter`：加载 openWakeWord model、读取 sounddevice 音频帧、调用 `Model.predict()`、生成 `WakeEvent`、维护 `WakeEngineStatus`。

adapter 不 import `voice_overlay.py`，不接触 command recorder，不调用回复链路。

## 3. Dependency Check 设计

`check_openwakeword_dependencies()` 在函数内部导入依赖，`openwakeword_adapter` 模块本身可以在缺少 openwakeword 时正常 import。

输出字段：

- `openwakeword_installed`
- `numpy_installed`
- `sounddevice_installed`
- `onnxruntime_available`
- `ready_for_realtime_demo`
- `errors`

`ready_for_realtime_demo=True` 只表示 adapter demo 依赖齐全，不表示小黄主链路已切换到专用 wake engine。

## 4. Adapter 生命周期

`start()`：

- 加载 numpy、openWakeWord model 和 sounddevice InputStream factory。
- 不在 import 阶段执行，不下载模型，不打开 STT server。
- 失败时把摘要写入 `status().error`。

`run_for_duration(duration_seconds, on_event=None, debug=False)`：

- 打开 sounddevice input stream，按 `chunk_ms` 读取 16kHz int16 mono frame。
- 调用 openWakeWord `Model.predict()`。
- 只对 `score >= sensitivity` 的 raw detection 进入 coalescer。
- 只对 coalesced wake event 调用 `on_event`。
- 到期退出；`KeyboardInterrupt` 会安全退出；`finally` 释放音频 stream 并 `stop()`。

`stop()`：

- 幂等，只切换 running 状态。
- `run_for_duration()` 的 stream 由 context manager 释放，不保留麦克风占用。

`status()`：

- 返回 `WakeEngineStatus(engine_type="openwakeword", running, ready, model_loaded, wake_phrase, sensitivity, last_wake_time, last_score, error)`。

## 5. 与 V1.2C Service 的关系

adapter 复用 V1.2C 的正式服务层：

- `WakeEvent`
- `WakeEngineStatus`
- `WakeEventCoalescer`
- `WakeEventStats`

cooldown 仍是 per-label：同一 label 在 cooldown 内只输出一个可消费 wake event，不同 label 不互相抑制。

## 6. 与 Demo 的关系

`scripts/wake_engine_demo.py` 的真实监听路径已改为走 `OpenWakeWordAdapter.run_for_duration()`。

保持兼容的 CLI：

## 9. 后续 V1.2D-B 接入前检查

进入主链路前还必须验证：

- wake listener 与 command recorder 的麦克风占用不会冲突。
- wake event 后可以暂停或释放 listener，再进入现有 VAD 命令录音。
- TTS 播放期间 pause / cooldown 生效，避免自唤醒。
- adapter 错误或模型缺失时可 fallback 到 `stt_text`。
- 控制面板如显示 wake engine 状态，不能混淆主链路 READY 与 wake engine READY。
