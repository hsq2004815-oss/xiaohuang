## Known Limitations

- Command recording captures ambient voices from nearby people. No speaker diarization.
- No near-field microphone lock.
- No weather tool — cannot query real-time weather.
- task_router is a safety placeholder; does not execute real tools.
- Not connected to: OpenCLI, opencode, browser automation, QQ, WeChat, crawlers, databases.
- Wake check is STT text-matching, not a dedicated KWS model.

## V1.1 Suggestions

1. Add wake-detected chime or more visible speaking/listening overlay state transitions
2. Optimize command recording window and VAD parameters to reduce ambient voice bleed
3. Implement overlay_controller_service.py before connecting real tools
