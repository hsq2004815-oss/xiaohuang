## 4. 修复过的验证 blocker

人工验证中发现并修复了一个 Settings UI 保存问题：

| 问题 | 根因 | 修复 |
|------|------|------|
| Advanced 页 `post_response_cooldown=None` 被保存为字符串 `"None"`，导致 reload / `--check` 失败 | Tkinter Entry 初始化使用 `str(None)`，保存层未把 `"None"` 还原为空值 | UI 显示 `None` 为空；保存层将空值 / `None` / `null` 规范为 JSON `null` |

修复后：

- `settings_ui.py --check` PASS
- 相关回归测试已加入
- 保存后的 `post_response_cooldown` 为 JSON `null`
