## 1. V1.1.3A 目标

把原来散落在以下各处的配置统一到一个中控层：

| 来源 | 原有问题 |
|------|---------|
| PowerShell 参数默认值 | `$Device = 0` 等隐形默认值覆盖 `config.json` |
| `argparse` 默认值 | `--wake-phrases default="小黄,小黄小黄"` 覆盖用户配置 |
| Python 常量 | `DEFAULT_TTS_VOICE` / `DEFAULT_WAKE_ALIASES` 散落各模块 |
| 旧 YAML config | `config_service.load_config` 返回 dict，与 dataclass 冲突 |

统一接入：`%USERPROFILE%\.xiaohuang\config.json` → `app_config_service.py` → `XiaoHuangConfig`

## 2. 已支持配置项

### 2.2 配置入口

| 入口 | 方式 | 说明 |
|------|------|------|
| `--config PATH` | CLI 参数 | 指定配置文件路径，默认 `%USERPROFILE%\.xiaohuang\config.json` |
| `-ConfigPath PATH` | PowerShell 参数 | `start_overlay.ps1` 透传 `--config` |
| `load_config(path)` | Python API | `app_config_service.load_config`，文件不存在时返回默认值 |

### 2.3 各配置项详细说明

**wake**
- `phrases`: 唤醒词列表。用户 config 中的值完全替换默认值 `["小黄"]`，不合并。
- `aliases`: 低置信度唤醒别名。空列表时不追加任何词。
- `wake_window_seconds`: 每次唤醒检测的录音窗口（秒）。

**audio**
- `device_id`: 麦克风设备 ID。PowerShell 只在 `$PSBoundParameters.ContainsKey("Device")` 时传 `--device`。
- `max_seconds`: 命令录音最大时长。
- `silence_seconds`: 语音结束后的静音截断阈值。

**llm**（预留，当前仅 deepseek 实际接入）
- `provider`: 提供商标识（`"deepseek"`）。
- `model`: 模型名。
- `base_url`: API 基础 URL。
- `timeout_seconds`: 请求超时。
- `max_tokens`: 最大 token 数。
- `temperature`: 生成温度。
- `api_key_env`: **环境变量名**，不存真实 key。

**tts**
- `enabled`: 是否启用 TTS 语音合成。
- `voice`: edge-tts 语音名。
- `output_dir`: TTS 输出目录（可选）。

**conversation**
- `enabled`: 是否启用多轮对话会话。
- `followup_timeout`: 每轮回复后的追问窗口（秒）。
- `max_turns`: 单次会话最大轮数。
- `max_session_seconds`: 单次会话最长时长（秒）。
- `max_no_speech_retries`: 连续无语音最大重试次数。
- `session_timeout`: 会话空闲超时（秒）。

**overlay**
- `resident_hidden`: 启动时隐藏悬浮窗，唤醒后显示。
- `post_response_cooldown`: 回复后的冷却时间（秒），`None` 时自动计算。

**runtime**
- `debug`: 是否打印调试输出。

**assistant**（V1.1.3A.4 新增）
- `name`: 助手身份名，用于 LLM 系统提示词。
- `display_name`: 悬浮窗标题显示名。
- `persona`: LLM 系统提示词全文，控制助手说话风格和行为。

## 3. 配置优先级

```
显式 CLI / PowerShell 参数  >  config.json  >  内置默认值
```

### 3.1 优先级实现

| 机制 | 函数 | 行为 |
|------|------|------|
| 用户 config merge | `merge_config_dict()` | JSON 字段覆盖默认 dataclass 字段 |
| CLI 覆盖 | `apply_cli_overrides()` | 显式传入的 CLI 值覆盖 config |
| `store_true` 布尔 | `_or_config(cli, config)` | `True` 覆盖 config，`False`（未传）回退 config |
| 标量值 | `_coalesce(cli, config)` | 第一个非 `None` 值生效 |
| wake phrases | `_coerce_phrases()` | 返回 `None` 时保留默认；非空列表时完全替换 |

### 3.2 `_or_config` 规则

```python
def _or_config(cli_value, config_value):
    if cli_value is True:   # 用户显式传了 --enable-llm
        return True
    return config_value      # 未传，回退 config 或默认
```

`store_true` 类型的参数（`--debug`、`--enable-llm`、`--enable-tts`、`--resident-hidden`、`--conversation-session`）：
- 命令行传了 → `True`，覆盖 config
- 命令行没传 → `False`（argparse 默认），此时**不回退为 False**，而是保留 config 值

### 3.3 PowerShell 边界

`start_overlay.ps1` 使用 `$PSBoundParameters.ContainsKey()` 判断是否显式传参：

| 参数 | 显式传入时 | 未传入时 |
|------|----------|---------|
| `-Device` | `--device $Device` | 不传 `--device`，config 生效 |
| `-EnableLlm` | `--enable-llm` | 不传，config 生效 |
| `-EnableTts` | `--enable-tts` | 不传，config 生效 |
| `-Debug` | `--debug` | 不传，config 生效 |
| `-ResidentHidden` | `--resident-hidden` | 不传，config 生效 |
| `-ConversationSession` | `--conversation-session` | 不传，config 生效 |
| `-ConfigPath "path"` | `--config path` | 不传，使用默认路径 |

## 4. Secrets / API Key 规则

### 4.1 设计原则

```
config.json → 只存 api_key_env（环境变量名）
secrets.ps1 → 设置 $env:DEEPSEEK_API_KEY = "sk-..."
环境变量   → process env，不落地文件
```

### 4.2 数据流

```
secrets.ps1  ──dot-source──→  $env:DEEPSEEK_API_KEY
                                 ↓
                          os.environ.get("DEEPSEEK_API_KEY")
                                 ↓
                          LlmReplyConfig(api_key=..., ...)
                                 ↓
                          Authorization: Bearer sk-...
```

### 4.3 安全规则（已实施）

- `LlmConfig` dataclass 无 `api_key` 字段，只有 `api_key_env: str`
- `_merge_llm()` 不读取 JSON 中的 `api_key` 键
- 日志不打印 `Authorization` header
- `_redact_url()` 函数脱敏 URL 中的 API key 参数
- `build_deepseek_response_debug_summary()` 不输出敏感内容
- `generate_llm_reply_result()` 只在 `on_debug` 回调存在时输出诊断，生产环境不输出

## 8. 当前禁止事项

以下方向不在 V1.1.3A 范围内，收口文档明确列出以防止范围蔓延：

1. **不做 HUD 美化** — 悬浮窗样式保持现状
2. **不做 Settings UI** — 配置通过文本编辑 `config.json`，不做图形设置面板
3. **不做 LLM Provider Router** — 当前只有 deepseek，多 provider 路由属于 V1.1.3B ✅ 已完成并真实验证通过（source=llm + TTS + Session）
4. **不做 Wake Engine Abstraction** — 当前唤醒逻辑在 `wake_loop_service.py`，不做引擎可插拔抽象
5. **不修改 STT/LLM/TTS 主链路** — 这些模块的输入输出协议不变
6. **不写 `E:\DataBase`** — 不在数据库目录写入文件
7. **不泄露 API key** — 不在 config/log/commit 中写入真实 key
