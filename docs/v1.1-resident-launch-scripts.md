# V1.1.1B — Resident Launch Scripts

> **Status:** Jarvis 常驻模式第一步 — 一键启动/停止/状态查看
> **Date:** 2026-05-01

## 目标

让用户可以通过一组 PowerShell 脚本一键启动/停止/查看小黄常驻模式。这是常驻桌面助手体验的基建，不包含系统托盘、开机自启动。

## 脚本

| 脚本 | 作用 |
|------|------|
| `scripts/start_stt_server.ps1` | 后台启动 STT server (`127.0.0.1:8766`)，自动检查是否已运行 |
| `scripts/start_overlay.ps1` | 后台启动 voice overlay，支持 `-Device` / `-EnableLlm` / `-EnableTts` / `-Debug` |
| `scripts/start_xiaohuang.ps1` | 一键启动：STT server → 等待 /health ready (最多 60s) → 启动 overlay |
| `scripts/status_xiaohuang.ps1` | 查看运行状态：/health 关键字段 + 进程列表 |
| `scripts/stop_xiaohuang.ps1` | 停止 overlay；`-StopSttServer` 同时停止 STT server |

## 使用方式

```powershell
cd E:\Projects\xiaohuang

# 完整启动（LLM + TTS + debug 日志）
.\scripts\start_xiaohuang.ps1 -Device 0 -EnableLlm -EnableTts -Debug

# 查看状态
.\scripts\status_xiaohuang.ps1

# 停止（保留 STT server）
.\scripts\stop_xiaohuang.ps1

# 停止全部
.\scripts\stop_xiaohuang.ps1 -StopSttServer
```

## 环境要求

- Python: `F:\for_xiaohuang\conda310\python.exe`（conda 3.10，已安装 FunASR）
- 模型缓存: `F:\for_xiaohuang\models\modelscope` / `F:\for_xiaohuang\models\huggingface`
- DEEPSEEK_API_KEY 由用户在终端自行设置，不写入任何脚本：
  ```powershell
  $env:DEEPSEEK_API_KEY="your key"
  ```
  如果未设置且使用 `-EnableLlm`，启动时会提示 "LLM may fallback to local rule replies"。

> 不要使用 base 环境 (`D:\miniconda3\python.exe`)。base 环境没有 FunASR，会报 `ModuleNotFoundError: No module named 'funasr'`。这是解释器选择错误，不是项目缺陷。

## 本地 Secrets 配置

每次启动时手动设置 API key 很麻烦。可以创建本地私有 secrets 文件，脚本会自动加载。

**创建 secrets 文件：**

```powershell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.xiaohuang" | Out-Null
notepad "$env:USERPROFILE\.xiaohuang\secrets.ps1"
```

**文件内容：**

```powershell
$env:DEEPSEEK_API_KEY="your_real_key_here"
```

- `secrets.ps1` 位于用户目录 `C:\Users\<you>\.xiaohuang\`，不在 Git 仓库内
- `.gitignore` 已排除 `.env` / `secrets.ps1` / `.xiaohuang/`
- 启动脚本会优先使用当前 shell 的环境变量；如果未设置则自动加载 secrets
- **绝对不要把真实 key 提交到 GitHub、写进脚本或文档**

以后直接运行即可，无需每次手动 set key：

```powershell
.\scripts\start_xiaohuang.ps1 -Device 0 -EnableLlm -EnableTts -Debug
```
