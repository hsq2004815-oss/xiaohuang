# V1.1.4 Resident / Tray / Launch Control 设计方案

> 阶段：V1.1.4A 设计文档
> 类型：docs-only
> 前置状态：V1.1.3A/B/C 已完成，Settings UI Prototype 已通过用户本机真人验证

## 1. V1.1.4 目标

V1.1.4 的目标是让小黄从“需要手动命令启动的语音助手”演进为“可由托盘管理的桌面常驻助手”。

第一版只做启动控制与常驻管理，不扩展语音能力。

必须覆盖：

- 托盘常驻。
- 从托盘启动小黄。
- 从托盘停止小黄。
- 从托盘重启小黄。
- 从托盘打开 Settings UI。
- 从托盘打开日志目录。
- 显示当前运行状态。
- 安全退出托盘程序。

当前已有启动链路应尽量复用：

```text
scripts/start_xiaohuang.ps1
  -> 启动 STT server
  -> 等待 /health
  -> 调用 scripts/start_overlay.ps1

scripts/start_overlay.ps1
  -> 加载 run_env.ps1
  -> 加载 %USERPROFILE%\.xiaohuang\secrets.ps1
  -> 后台启动 scripts/voice_overlay.py

scripts/stop_xiaohuang.ps1
  -> 停止小黄相关进程

scripts/settings_ui.py
  -> 打开配置编辑器

scripts/voice_overlay.py
  -> Tkinter 语音悬浮窗主入口
```

## 2. 非目标

V1.1.4 不做以下内容：

- 不做高级 HUD。
- 不做复杂悬浮窗视觉升级。
- 不做开机自启。
- 不做安装器。
- 不做工具调用。
- 不接浏览器 / QQ / 微信。
- 不接 OpenCLI / opencode。
- 不做爬虫。
- 不做插件系统。
- 不改 wake engine。
- 不改 LLM Provider Router。
- 不改 conversation session 行为。
- 不保存真实 API key。

## 3. 推荐新增文件

### 3.1 `scripts/tray_app.py`

托盘入口脚本，负责：

- 创建 Windows 托盘图标。
- 构建托盘菜单。
- 调用启动 / 停止 / 重启脚本。
- 打开 Settings UI。
- 打开日志目录。
- 查询并展示运行状态。
- 写托盘自身日志。

建议保持这个脚本为 UI/编排层，不直接塞入复杂进程识别逻辑。

### 3.2 可选：`src/xiaohuang/process_status_service.py`

进程状态检测服务，负责：

- 检查 STT server 是否健康。
- 检查 `voice_overlay.py` 是否存在。
- 检查 PowerShell 启动链路是否已运行。
- 返回结构化状态，例如：

```text
stt_server: running / stopped / unknown
overlay: running / stopped / unknown
settings_ui: running / stopped / unknown
overall: ready / partial / stopped / error
```

### 3.3 可选：`src/xiaohuang/launch_control_service.py`

启动控制服务，负责：

- 统一封装 start / stop / restart。
- 通过 `subprocess` 调用现有 PowerShell 脚本。
- 处理路径、编码、超时、返回码。
- 避免托盘 UI 直接拼复杂 PowerShell 命令。

## 4. 推荐不修改或少改的文件

这些文件已经承担主链路职责，V1.1.4 不应扩大修改面：

- `scripts/voice_overlay.py`
- `src/xiaohuang/llm_reply_service.py`
- `src/xiaohuang/wake_loop_service.py`
- `src/xiaohuang/conversation_session_service.py`
- `src/xiaohuang/settings_config_file_service.py`

只有在发现托盘启动暴露出明确集成 bug 时，才考虑最小修复；否则 V1.1.4 应保持主链路稳定。

## 5. 托盘菜单设计

推荐菜单：

```text
小黄
├─ 状态：已运行 / 未运行 / 部分运行 / 异常
├─ 启动小黄
├─ 停止小黄
├─ 重启小黄
├─ 打开设置
├─ 打开日志目录
├─ 查看状态
└─ 退出托盘
```

菜单行为：

| 菜单项 | 行为 |
|--------|------|
| 启动小黄 | 调用 `scripts/start_xiaohuang.ps1` |
| 停止小黄 | 调用 `scripts/stop_xiaohuang.ps1 -StopSttServer` |
| 重启小黄 | 先停止，再启动 |
| 打开设置 | 启动 `scripts/settings_ui.py` |
| 打开日志目录 | 打开 `E:\Projects\xiaohuang\logs` |
| 查看状态 | 弹出简短状态消息 |
| 退出托盘 | 退出托盘程序；是否同时停止小黄需明确策略 |

建议第一版“退出托盘”默认只退出托盘，不停止小黄；菜单另有“停止小黄”负责停服务。这样更符合“托盘管理器”和“助手运行进程”分离的边界。

## 6. 进程管理方案

### 6.1 如何识别 STT server

优先使用 HTTP health check：

```text
GET http://127.0.0.1:8766/health
```

判断：

- 返回 ready / ok：STT server running。
- 连接失败：STT server stopped。
- 超时或异常：STT server unknown/error。

不要仅凭进程名判断 STT server，因为同一 Python 环境可能运行其他脚本。

### 6.2 如何识别 `voice_overlay`

可组合使用：

- `Win32_Process.CommandLine` 中包含 `scripts\voice_overlay.py`。
- 工作目录或命令行包含 `E:\Projects\xiaohuang`。

避免只按 `python.exe` 进程名识别。

### 6.3 如何避免重复启动

启动前先查询：

- STT server `/health` 是否 ready。
- `voice_overlay.py` 是否已存在。

如果两者都运行：

- 托盘显示“已运行”。
- 点击“启动小黄”只提示“已运行”，不重复启动。

如果只有部分运行：

- 可提示“检测到部分运行，建议点击重启小黄”。
- V1.1.4C 可以选择先 stop 再 start。

### 6.4 如何停止子进程

优先复用：

```powershell
.\scripts\stop_xiaohuang.ps1 -StopSttServer
```

理由：

- 现有脚本已经是小黄进程停止入口。
- 保持停止逻辑集中，避免托盘程序重复实现进程杀除规则。
- 后续如果停止策略变化，只改脚本或 launch service。

### 6.5 是否复用现有 PowerShell 脚本

建议复用。

V1.1.4 托盘程序应通过 `subprocess` 调用：

```text
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts/start_xiaohuang.ps1
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts/stop_xiaohuang.ps1 -StopSttServer
```

可选传入：

```text
-ConfigPath "%USERPROFILE%\.xiaohuang\config.json"
```

### 6.6 是否通过 `subprocess` 调用 PowerShell

建议是。

注意：

- 指定 `cwd=E:\Projects\xiaohuang`。
- 捕获 stdout/stderr 写入托盘日志。
- 使用 UTF-8 环境变量，减少中文乱码。
- 调用 start 时不能阻塞托盘 UI 线程，应放后台线程。
- 设置合理超时；启动脚本可能需要等待模型或 STT server health。

## 7. 配置路径方案

默认配置路径：

```text
%USERPROFILE%\.xiaohuang\config.json
```

V1.1.4A/B 可先不做复杂配置选择。

未来可支持：

```text
scripts/tray_app.py --config "%USERPROFILE%\.xiaohuang\config.json"
```

边界：

- 托盘程序不保存真实 API key。
- 托盘程序只传递 `ConfigPath`。
- 真实 key 仍通过 `%USERPROFILE%\.xiaohuang\secrets.ps1` 或环境变量加载。
- 不在托盘日志中打印真实 key 或完整环境变量值。

## 8. 日志方案

### 8.1 托盘自身日志

建议新增：

```text
logs/tray_app.out.log
logs/tray_app.err.log
```

记录：

- 托盘启动时间。
- 菜单操作：start / stop / restart / open settings / open logs。
- PowerShell 返回码。
- 简短错误原因。
- 状态检测摘要。

不要记录：

- 真实 API key。
- 完整 secrets.ps1 内容。
- 录音文本敏感内容。

### 8.2 打开 logs 目录

菜单“打开日志目录”直接打开：

```text
E:\Projects\xiaohuang\logs
```

Windows 上可以使用：

```text
os.startfile(LOG_DIR)
```

或通过 PowerShell：

```powershell
explorer.exe .\logs
```

### 8.3 错误提示

出错时先用简短 `messagebox`：

```text
启动失败，请查看 logs/tray_app.err.log
```

不要在第一版弹复杂诊断窗口。

## 9. 风险点

| 风险 | 说明 | 缓解 |
|------|------|------|
| `pystray` / `Pillow` 新依赖 | 托盘图标通常依赖这两个包 | V1.1.4B 先确认环境安装成本；不盲目加依赖 |
| Windows 后台进程识别 | 多个 `python.exe` 容易误判 | 用 command line + health check 组合识别 |
| PowerShell 子进程编码 | 中文输出可能乱码 | 设置 UTF-8 环境，日志用 UTF-8 写入 |
| 重复启动 | 多次点击可能启动多个 overlay | 启动前状态检测，操作按钮短暂禁用 |
| 退出托盘是否停止小黄 | 用户预期可能不同 | 第一版明确“退出托盘不停止小黄”，保留“停止小黄”菜单 |
| 用户手动关闭 overlay 后状态刷新 | STT server 可能仍在运行，overlay 已停止 | 状态显示 `partial`，建议重启 |
| STT server 启动慢 | 模型初始化和 health 等待需要时间 | 菜单提示“启动中”，后台线程处理 |
| Settings UI 多开 | 多次打开可能出现多个配置窗口 | V1.1.4B 可暂不阻止，V1.1.4D 再做状态检测 |

## 10. 验证方案

自动验证：

```powershell
& "F:\for_xiaohuang\conda310\python.exe" -m unittest discover -s tests
& "F:\for_xiaohuang\conda310\python.exe" -m compileall -q src scripts tests
& "F:\for_xiaohuang\conda310\python.exe" scripts\tray_app.py --help
```

人工验证：

1. 启动托盘程序。
2. 确认托盘图标出现。
3. 点击“打开设置”，Settings UI 正常打开。
4. 点击“打开日志目录”，`logs/` 正常打开。
5. 点击“查看状态”，状态提示可读。
6. 点击“启动小黄”，STT server 和 overlay 正常启动。
7. 再次点击“启动小黄”，不重复启动。
8. 真实语音唤醒仍正常。
9. 点击“停止小黄”，STT server 和 overlay 停止。
10. 点击“重启小黄”，可恢复运行。
11. 点击“退出托盘”，托盘程序退出。

日志检查：

```powershell
Get-Content .\logs\tray_app.err.log -Tail 300
Get-Content .\logs\voice_overlay.err.log -Tail 500 |
  Select-String "Traceback|ERROR|HTTPError|TypeError|UnboundLocalError"
```

预期：

- 托盘日志无 traceback。
- 小黄主链路日志无新增异常。
- 真实语音唤醒、LLM、TTS、session exit 仍保持 V1.1.3C 验证结果。
