# V1.2D-B Wake Engine 安全验证

状态：安全验证入口已新增；未接入正式 `voice_overlay.py` 唤醒链路
日期：2026-05-03
范围：OpenWakeWordAdapter 麦克风生命周期、重复启动/停止、异常释放和 callback 合并验证

## 1. 目标

V1.2D-B 的目标是在接入主链路前验证 adapter 是否能安全地启动、停止并释放麦克风资源，避免后续与现有 VAD command recorder、TTS 播放和 STT text fallback 产生冲突。

本阶段不修改 `scripts/voice_overlay.py`，不替换 `wake_loop_service.py` / `wake_word_service.py` 的 STT 文本唤醒，不训练中文“贾维斯”模型，不新增依赖。

## 2. 为什么先做安全验证

openWakeWord 的 raw detection 是连续帧级命中。正式接入后，如果 wake listener 不释放麦克风，现有命令录音会抢不到设备；如果 cooldown 或 callback 边界不清晰，一次唤醒可能触发多次命令录音；如果 TTS 播放期间仍监听，可能出现自唤醒。

因此 D-B 只验证 adapter 和 demo harness 的生命周期，不把事件送入主链路。

## 3. 本次验证内容

- `OpenWakeWordAdapter.stop()` 在 start 前、运行后和重复调用时保持幂等。
- `run_for_duration()` 在正常结束、普通异常和 `KeyboardInterrupt` 时都会离开 stream context 并 `stop()`。
- `status().model_loaded` 与 `status().ready` 分离：模型已加载后即为 `model_loaded=True`，运行错误只影响 `ready/error`。
- 只对 coalesced wake event 调用 callback，cooldown 内 suppressed detection 不调用 callback。
- `scripts/wake_engine_demo.py` 新增 `--safety-check`，支持重复 start/run/stop 轮次。

## 4. 安全边界

- 不启动 STT server。
- 不启动 `voice_overlay.py`。
- 不调用 LLM、TTS 或 reply pipeline。
- 不改 PowerShell 启停脚本、控制面板或托盘。
- 不写运行日志、音频、模型缓存或 `E:\DataBase`。
- 单元测试使用 fake model、fake stream 和 fake adapter，不打开真实麦克风。

## 6. 期望输出

每轮应输出：

```text
round=<n> started=true
round=<n> stopped=true
round=<n> frames=<n>
round=<n> raw_detections=<n>
round=<n> coalesced_events=<n>
round=<n> suppressed_detections=<n>
round=<n> status_after_stop running=false ready=false model_loaded=true error=-
```

最终应输出：

```text
safety_check=true
rounds=2
all_rounds_completed=true
microphone_released=true
errors=0
```

`microphone_released=true` 表示 adapter 已离开 stream context 且 `status_after_stop.running=false`；它不是 Windows 音频设备层面的独占锁证明。真实麦克风占用仍建议人工短测。

## 8. 风险

- sounddevice / Windows driver 仍可能在真实设备上出现 busy、overflow 或释放延迟。
- 当前验证不进入 command recorder，因此只能证明 adapter 自身会停止，不证明与 VAD 录音的真实切换体验。
- 当前 `wake_phrase=贾维斯` 仍是显示名，真实可用模型仍是英文 `hey_jarvis` smoke，不代表中文模型完成。
- TTS 自唤醒仍需在主链路接入设计中通过 pause/cooldown/mute window 处理。

## 9. 回滚

D-B 没有修改主链路。回滚时移除 `wake_engine_demo.py --safety-check` 入口和相关测试即可；现有 `stt_text` wake path、控制面板、托盘、PowerShell、LLM/TTS 均不受影响。
