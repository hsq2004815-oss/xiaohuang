# XiaoHuang Agent Guide

## Role And Scope

This repository contains XiaoHuang V0.7, a minimal Windows audio pipeline validation project.

The current scope is intentionally narrow:

- enumerate Windows microphone devices
- record a fixed-duration WAV sample
- save recordings under `data/recordings/`
- stop `listen_once.py --vad` recordings with a lightweight energy-threshold VAD after speech and silence
- run `wake_loop.py` as a console-only STT text-matching wake prototype
- run `voice_overlay.py` as a Tkinter always-on-top voice overlay prototype
- delete wake-check short recording temp files by default unless `--keep-wake-recordings` is used
- restrict STT server `/transcribe` reads to existing `.wav` files under project `data/recordings/`
- transcribe a WAV file with FunASR / SenseVoiceSmall when the STT dependencies are installed
- print results to the console and write script logs under `logs/`
- run `listen_once.py` to record once, transcribe once, and print timing diagnostics
- run `stt_server.py` locally on `127.0.0.1` to keep SenseVoiceSmall loaded across requests

## Hard Boundaries

Do not add these features in V0.7:

- TTS
- OpenCLI integration
- opencode integration
- QQ / WeChat integration
- browser automation
- crawlers or data downloaders
- full task scheduling or agent orchestration
- FunASR KWS integration
- openWakeWord training or custom model work
- installer or packaging system

V0.7 wake handling is still only an STT text-matching prototype displayed in Tkinter. Do not train or add a real openWakeWord/FunASR KWS model unless explicitly requested.

Do not modify `E:\DataBase` from this repository work. Database files are read-only context.

## Development Rules

- Keep changes small and focused on the minimum audio path.
- Prefer project-local virtual environments.
- Do not install packages globally.
- Do not commit `.venv`, WAV recordings, logs, model caches, or generated bytecode.
- Keep `README.md` and `MEMORY.md` updated when behavior or setup changes.
- For the validated local V0.7 environment, prefer `F:\for_xiaohuang\conda310\python.exe` and dot-source `.\scripts\run_env.ps1` before manual audio checks.
- `run_env.ps1` is a print-only helper; do not change it to auto-record or auto-transcribe without an explicit request.
